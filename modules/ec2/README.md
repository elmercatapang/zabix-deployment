# Module: ec2

Provisions the Zabbix server EC2 instance, Elastic IP, TLS key pair, and runs the Zabbix installation via EC2 user data.

## Resources Created

| Resource | Terraform ID | Name (tag) | Description |
|---|---|---|---|
| `aws_instance` | `zabbix` | `<name>-zabbix-server` | EC2 instance running Ubuntu 22.04 LTS |
| `aws_eip` | `zabbix` | `<name>-zabbix-eip` | Elastic IP attached to the instance |
| `aws_key_pair` | `zabbix` | `<name>-zabbix-key` | EC2 key pair (public key uploaded to AWS) |
| `tls_private_key` | `zabbix` | — | RSA 4096-bit key generated by Terraform |
| `local_sensitive_file` | `private_key` | — | Private key written to `zabbix-<name>.pem` (mode 0600) |

## AMI

Ubuntu 22.04 LTS (Jammy), HVM, most recent — sourced from Canonical's AWS account (`099720109477`). Resolved dynamically at plan time.

## User Data — What Gets Installed

The `user_data.sh.tpl` script runs on first boot and installs the full Zabbix stack. Install log is written to `/var/log/zabbix-install.log`.

| Step | Detail |
|---|---|
| System update | `apt-get update && apt-get upgrade` |
| PostgreSQL | Installed from Ubuntu repos; DB and user created at boot |
| Zabbix 7.0 LTS | Installed from the official Zabbix APT repo (`repo.zabbix.com`) |
| Components installed | `zabbix-server-pgsql`, `zabbix-frontend-php`, `zabbix-nginx-conf`, `zabbix-sql-scripts`, `zabbix-agent2` |
| Schema import | `/usr/share/zabbix-sql-scripts/postgresql/server.sql.gz` |
| Zabbix server config | `/etc/zabbix/zabbix_server.conf` — DBName, DBUser, DBPassword set |
| Nginx config | `/etc/zabbix/nginx.conf` — listen 80, server_name `_` |
| Zabbix agent2 | Configured to monitor the server itself (Server=127.0.0.1) |
| Services enabled | `zabbix-server`, `zabbix-agent2`, `nginx`, `php8.1-fpm` |
