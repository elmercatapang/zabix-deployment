# Zabbix Deployment

Terraform project to deploy a Zabbix 7 LTS monitoring server on AWS EC2. Replaces PRTG. Covers two isolated environments that share the same reusable module set.

## Repository Structure

```
zabbix-deployment/
├── modules/
│   ├── vpc/        VPC, subnets, IGW, route tables
│   ├── sg/         Security group (web UI, SSH, Zabbix agents)
│   └── ec2/        EC2 instance, EIP, key pair, user data install script
└── environments/
    ├── sandbox/    AWS profile: ai-dev    | Account: 145770591346 | CIDR: 172.16.0.0/16
    └── prod/       AWS profile: infra-secops               | CIDR: 172.17.0.0/16
```

## Deploy — Sandbox First

```bash
cd environments/sandbox
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars: set db_password and ssh_allowed_cidrs
terraform init
terraform plan
terraform apply
```

After apply, Terraform outputs:
- `zabbix_public_ip` — Elastic IP of the server
- `zabbix_url` — `http://<ip>/zabbix` (default credentials: Admin / zabbix — change immediately)
- `ssh_command` — ready-to-run SSH command
- `private_key_file` — path to the generated `.pem` file

## Deploy — Production

```bash
cd environments/prod
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars
terraform init

```

## Security Notes

- SSH key pair is generated by Terraform and written locally as `zabbix-<env>.pem` (mode 0600, gitignored)
- `terraform.tfvars` is gitignored — never commit it
- Root EBS volume is encrypted at rest
- Change the default Zabbix admin password (`Admin` / `zabbix`) immediately after first login
